bool Plateau::affichageIntegraleActif ( ) const

Indique si les prix sont à afficher en intégralité, ou en forme compressée.

Si cette méthode retourne true, les prix sont à afficher intégralement. Par exemple, avec un coefficient très gros :

  • 1 500 000 000;
  • 20 000 000 ;
  • 14 500 000 ;
  • 500 000.

Si cette méthode retourne false, les prix sont à afficher avec des lettres. Par exemple, avec les prix ci-dessus :

  • 1 500M ;
  • 20M ;
  • 14,5M;
  • 500k.
Renvoie:
true si les nombres sont à afficher en intégralité.

bool Plateau::m_prixAffichageComplet [private]

Affichage complet des prix.

Si cet attribut vaut true, les prix sont affichés intégralement. Par exemple, avec un coefficient très gros :

  • 1 500 000 000;
  • 20 000 000 ;
  • 14 500 000 ;
  • 500 000.

Si cet attribut vaut false, les prix sont affichés avec des lettres. Par exemple, avec les prix ci-dessus :

  • 1 500M ;
  • 20M ;
  • 14,5M;
  • 500k.
quint8 Plateau::m_prixCoefficient [private]

Coefficient utilisé pour les prix.

Peut aller de 0 à 8. Le coefficient réel vaut 10^m_nombreCoefficient. Par exemple, si m_prixCoefficient vaut 2, le coefficient est de 100, si m_prixCoefficient vaut 5, le coefficient est de 100 000.

En interne, les prix sont enregistrés dans des quint16 et doivent être compris entre 0 et 50 000. Il est donc possible d'appliquer un coefficient à tous les prix afin d'obtenir 20M € (au lieu de 200 €) en passant par la case Départ par exemple.
